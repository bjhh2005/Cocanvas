## 一、 整体路由与视图结构

前端只包含两个核心路由，保持极简的流转体验：

* **`/` (首页)**：系统的入口，主要承载创建房间的功能。
* **`/room/:roomId` (协作工作区)**：系统的核心视图，承载全屏白板、工具栏和历史回放。

---

## 二、 首页界面设计 (Home Page)

首页采用极简的居中卡片布局，重点突出“快速开始”的动作。

### 视觉布局

**背景层**
应用浅灰色的网格背景（模拟无限画布的视觉暗示），辅以缓慢漂浮的几个彩色半透明几何图形（矩形、圆形），营造协同白板的氛围。

**居中操作卡片**

* **Logo与标题**：顶部居中显示 “Cocanvas” 标题，副标题为 “Distributed Real-time Whiteboard”。
* **主操作区**：一个醒目的主按钮，文案为“创建新房间 (Create Room)”。按钮采用品牌主色（如深蓝色或亮紫色），带有微小的悬浮阴影放大动效。
* **辅助操作区**：主按钮下方是一个包含输入框和次级按钮的组合。输入框提示词为“输入房间 ID (Room ID)”，右侧紧跟“加入 (Join)”按钮，供已知房间号的用户快捷进入。

### 交互反馈

点击“创建新房间”时，按钮呈现 Loading 态。调用 `POST /api/rooms` 成功并获取到 `roomId` 与 `wsUrl` 后，卡片整体产生向上淡出的退场动画，页面无缝路由推入 `/room/:roomId`。

## 三、 协作工作区界面设计 (Room Page)

这是系统的绝对核心。整个视口（100vh, 100vw）被划分为四大视觉层：**底层画布区**、**顶部状态控制栏**、**左侧绘图工具栏**、**底部时光机控制区**。

### 1. 底层画布区 (Canvas Area)

占据 100% 屏幕空间，无滚动条（通过鼠标拖拽和滚轮实现无限平移与缩放）。

* **网格背景**：浅灰色的点阵或网格线，辅助用户对齐图形。
* **图形层 (Shape Layer)**：渲染所有 CRDT 状态下的图形实体（矩形、圆形、文本、连线、图片）。选中本地图形时，图形四周会出现蓝色的边界框（Bounding Box）和缩放控制手柄。
* **协同光标层 (Cursor Layer)**：绝对定位的 `div` 元素层，覆盖在 Konva 画布之上。每个光标由一个带有特定颜色（服务端下发）的 SVG 鼠标指针图标和一个跟随其后的圆角文字标签（用户昵称）组成。光标移动采用 CSS `transform` 与 `transition: transform 50ms linear` 实现丝滑移动。

### 2. 顶部状态控制栏 (Header & Status Bar)

悬浮在视口顶部，采用半透明毛玻璃效果（Backdrop-filter），左右分列。

* **左侧：房间信息**
包含返回首页的轻量 Icon，以及当前房间的 ID 显示（附带“一键复制”小图标）。
* **中间：历史回放入口**
一个带有类似“时钟”图标的按钮，文案为“时光回放 (Time Machine)”。点击后将触发底部的回放控件展开。
* **右侧：连接与在线状态**
* **连接指示器**：一个小圆点加文本。绿色代表 `Connected`，黄色代表 `Connecting/Reconnecting`，红色代表 `Disconnected`。断网或切换节点触发重连时，这里会闪烁并提示“正在连接至新节点...”。
* **在线头像组**：显示当前房间内所有用户的彩色圆点头像（显示昵称首字母）。当有人加入或退出时，头像组产生平滑的挤压排布动画。



### 3. 左侧绘图工具栏 (Floating Toolbar)

悬浮在视口左侧垂直居中位置的胶囊状工具面板，包含图标按钮。选中不同工具时，对应图标高亮。

* **基础工具**：选择指针（Select）、拖拽手势（Pan - 纯移动画布）。
* **创建工具**：矩形（Rect）、圆形（Circle）、文本（Text）、连线（Line - 点击两图形建立关联）、图片（Image）。
* **属性控制（条件弹窗）**：当选中某个图形时，工具栏右侧会滑出一个子面板，提供颜色选择器（填充色、描边色）和线条粗细调节。
* **删除动作**：独立的垃圾桶图标按钮（也可通过键盘 Delete 键触发）。

### 4. 底部时光机控制区 (Timeline Slider)

默认隐藏，点击顶部“时光回放”后从底部滑出，并为整个页面蒙上一层极淡的暗色遮罩，提示进入了“只读模式”。

* **时间轴滑块**：横跨屏幕底部的 Range Input。左侧极点是“房间创建时间”，右侧极点是“当前实时”。
* **时间刻度与快照标识**：滑块轨道上带有微小的竖线标记，代表服务端已持久化的快照（Snapshot）点。
* **时间显示器**：滑块上方悬浮显示当前拖动位置对应的时间戳（如 `2023-10-25 14:30:22`）。
* **退出回放按钮**：位于控制区右上角的一个醒目“退出”按钮，点击后销毁回放画布，返回实时协作层。

## 三、 画布与光标系统的深度设计

### 1. 多人光标层 (CursorLayer)

这是展现“协作感”最直观的地方。

* **DOM 渲染机制**：不使用 Konva 渲染光标（因为光标更新极快，DOM 渲染性能更好且不影响画布重绘）。在 `<Stage>` 之上叠加一个绝对定位的 `<div id="cursors">`。
* **单光标视觉**：一个 SVG 箭头，填充色为后端广播过来的该用户的专属颜色。箭头右下角紧贴一个带有该颜色的圆角小标签，显示 `displayName`。
* **平滑动效**：因为前端 `onMouseMove` 是节流发送（50ms），接收端如果直接设置 `x/y` 会显得卡顿。必须在光标 DOM 上添加 CSS 过渡：`transition: transform 0.08s linear`，利用浏览器 GPU 进行补间动画，实现丝滑跟随。

### 2. CRDT 冲突的视觉处理

当发生并发拖拽冲突时，你的 HLC + LWW 算法在后端或本地 store 算出最终赢家。

* **UI 表现**：不需要做复杂的合并 UI。图形会因为 HLC 胜出者的状态，发生一次“瞬移”（Snapping）。为了让这种“瞬移”不那么突兀，可以在 Konva 的 Shape 组件上开启轻微的缓动（Tween），当属性突变时，用 100ms 的时间平滑过渡到新位置或新颜色。

## 四、 时光机：历史回放视图 (Time Machine)

这是答辩和展示架构能力的“杀手锏”功能，UI 必须做到一目了然，防止用户将历史状态与实时状态混淆。

### 1. 触发与视觉模式切换

* 用户点击顶部栏的 “历史回放” 按钮。
* **界面异化**：整个屏幕四周出现一圈 4px 的显眼边框（例如警戒橙色），并在右上角打上巨大的半透明水印字样 **"READ ONLY - HISTORY"**。
* 同时，左侧工具栏隐藏，悬浮属性面板禁用，屏蔽一切 Konva 的交互事件。

### 2. 底部时间轴滑块 (Timeline Slider)

屏幕底部升起一个贯穿全宽的控制面板。

* **UI 结构**：
* 左端点：房间创建时间。
* 右端点：当前最新时间。
* 中央主体：一个宽大的 `<input type="range">` 滑块组件。
* 当前戳提示：滑块手柄上方悬浮显示当前拨到的具体时间（精确到秒）。


* **交互机制**：
* 当滑块被拖动（`onChange`）时，使用 `lodash.debounce` 防抖（约 200ms）触发向后端的 `GET /api/rooms/{roomId}/history?at=...` 请求。
* 获取到 Snapshot 和 Ops 后，推入独立的**只读 Store**，Konva 画布重新渲染历史状态。


* **退出机制**：面板正中央上方提供一个醒目的 `[退出时光机，回到现在]` 按钮，点击后恢复普通模式的 Store 并卸载只读样式。

## 五、 容错与异常反馈设计

分布式系统极其考验异常情况下的前端表现：

1. **节点故障切换 (Failover) 的 UI 表现**：
* 当用户正在拖拽图形，如果连接到底层的 WebSocket 节点被 kill 掉，拖拽中的图形应立即停留在当前位置（本地状态）。
* 顶部栏瞬间变成 🟡 **"重连中 (尝试 1/5)..."**。此时禁用左侧工具栏，防止用户产生新的未确认操作。
* 10 秒内，底层通过 `ConsistentHashRing` 拿到新节点 WS URL 并连上，拉取最新 Snapshot。画布可能会发生一次轻微的闪烁（状态对齐），顶部栏恢复 🟢 **"已连接"**，工具栏解锁。整个过程透明、免刷新。


2. **操作丢失补偿**：
* 为了极致的体验，可以在前端给未收到后端 Ack（自己发出的 Op 广播被自己收到）的图形加上极微弱的透明度（`opacity: 0.8`）。一旦确认同步，恢复 `1.0`。如果重连后发现某些操作没报上去，前端可以静默丢弃并按服务端的 Snapshot 渲染，保证最终一致性。
