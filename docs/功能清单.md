# 功能清单

> 更新日期: 2026-05-30  
> 说明: 本文档描述当前代码已经具备的能力和仍然存在的限制。旧编号文档保留为历史设计材料, 当前功能状态以本文档为准。

## 1. 产品定位

Cocanvas 当前是一个多人实时协作白板工作台, 同时用于展示分布式协作系统的关键机制。

核心使用路径:

1. 用户进入房间控制台。
2. 创建或加入房间。
3. 在房间中多人协作编辑白板。
4. 系统通过 WebSocket 实时同步操作。
5. 断线或节点故障后, 客户端重连并通过历史恢复重建状态。

## 2. 房间能力

已完成:

- 房间列表。
- 创建房间。
- 自定义房间号。
- 自定义房间名称。
- 房间访问模式字段:
  - `link`
  - `password`
  - `private`
- 默认权限字段:
  - `edit`
  - `comment`
  - `view`
- 房间密码进入校验。
- 更新房间设置。
- 归档房间。
- 查询房间是否存在。
- 返回当前房间应连接的 WebSocket 地址。
- 会议语音入口开关字段。

已知限制:

- `permissionMode` 目前主要是元数据, 后端尚未基于它拒绝 op 写入。
- `private` 尚未实现成员邀请和访问控制。
- 密码实现仍偏开发级, 后续应改为带盐哈希和短期 join token。

## 5. 实时协作

已完成:

- Spring WebSocket 原生协议。
- 房间 join。
- `joined` 回执。
- `peer-joined` / `peer-left`。
- 远端光标广播。
- shape operation 广播。
- shape preview 广播。
- 本地先应用 op, 服务端持久化成功后 ack。
- 发送方收到 `op-ack` 后从 pending outbox 删除。
- 收到 `op-ack` 时使用服务端合并后的 HLC 校准本地版本。
- WebSocket 断线重连。
- 重连后重新 join。
- 重连后恢复历史状态。
- 断线期间 pending op 补发。

已知限制:

- `RoomSessionRegistry.broadcastInRoom` 仍是同步逐 session 发送, 尚未实现慢客户端背压隔离。
- cursor 和 shape-preview 仍会跨节点广播, 尚未拆为低可靠或可丢弃通道。
- WebSocket join 尚未 token 化。

## 6. 分布式能力

已完成:

- Docker Compose 启动双后端节点:
  - `backend1`
  - `backend2`
- Nginx 统一入口。
- 节点专属 WebSocket 路径:
  - `/ws/backend1/collab`
  - `/ws/backend2/collab`
- Redis 节点心跳。
- Redis `cocanvas:nodes` Set 维护节点索引。
- 节点 TTL key 表示存活状态。
- 一致性哈希按 `roomId` 选择 owner 节点。
- 一致性哈希环缓存。
- Redis Pub/Sub 跨节点广播。
- Pub/Sub 频道按 hash 分为 64 个:
  - `cocanvas:room-events:0`
  - ...
  - `cocanvas:room-events:63`
- Redis 订阅端将远端 op 应用到本节点内存副本。
- 只有一致性哈希 owner 节点写 room snapshot。

已知限制:

- 节点列表变化后, 已连接 WebSocket 需要依赖断线重连迁移。
- 没有专门的节点负载指标和动态权重。
- Redis Pub/Sub 不提供消息持久化, op 可靠性依赖 MySQL 操作日志和客户端重放。

## 7. 一致性与历史恢复

已完成:

- HLC 时间戳。
- 属性级 LWW/CRDT 合并。
- delete tombstone。
- opId 幂等去重。
- MySQL 操作日志。
- MySQL snapshot。
- versioned snapshot:
  - 每个属性保留 `value`
  - 每个属性保留 `hlc`
  - 每个属性保留 `writerId`
- 历史恢复优先读取目标时间之前最新 snapshot。
- 只回放 snapshot 之后的增量 op。

已知限制:

- 历史回放和实时 store 还没有完全隔离。
- 缺少专门的历史时间轴 UI。
- 缺少 snapshot 完整性校验字段。

## 9. 下一步建议

P0:

- 实现 WebSocket 慢客户端背压:
  - 每个 session 独立发送队列。
  - cursor / preview 超限丢弃。
  - op 超限断开并要求重连恢复。
- 实现后端权限校验:
  - `view` 用户不能发送 op。
  - `comment` 用户只能发 comment 类操作。
- 引入 join token。

P1:

- cursor / shape-preview 拆为低优先级可丢弃通道。
- 房间元数据加 Redis 或本地缓存, 降低高并发 REST 查询 p99。
- 历史回放和实时状态拆 store。
- 增加协作诊断面板:
  - 当前节点
  - wsUrl
  - pending ops
  - reconnect 次数
  - last restored ops

P2:

- WebRTC 语音协作。
- 白板对象虚拟化。
- 完整权限成员模型。
- Flyway/Liquibase 数据库迁移。
