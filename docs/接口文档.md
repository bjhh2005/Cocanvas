# 接口文档

> 更新日期: 2026-05-30  
> 说明: 本文档描述当前代码实际实现的 REST 和 WebSocket 协议。旧 `05_接口文档.md` 保留为历史设计材料, 当前接口以本文档为准。

## 1. 通用约定

基础地址:

```text
HTTP:      http://localhost:8088/api
WebSocket: ws://localhost:8088/ws/backend1/collab
WebSocket: ws://localhost:8088/ws/backend2/collab
```

开发兼容路径:

- 多数 REST 接口同时兼容去掉 `/api` 前缀的路径。
- 通用 WebSocket 路径 `/ws/collab` 仍保留。

数据格式:

- JSON。
- UTF-8。
- 字段使用 camelCase。
- 时间为毫秒时间戳。
- HLC 格式为 `physicalMs.logicalCounter.nodeId`, 示例: `1780123820639.1501.node-local`。

## 2. REST 接口

### 2.1 健康检查

```http
GET /api/health
```

响应:

```json
{
  "status": "ok"
}
```

### 2.2 创建房间

```http
POST /api/rooms
Content-Type: application/json
```

请求:

```json
{
  "roomId": "demo1234",
  "name": "Demo room",
  "accessMode": "link",
  "permissionMode": "edit",
  "password": "",
  "voiceEnabled": false
}
```

字段:

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `roomId` | string | 否 | 不传则后端生成短 ID |
| `name` | string | 否 | 房间名称 |
| `accessMode` | string | 否 | `link` / `password` / `private` |
| `permissionMode` | string | 否 | `edit` / `comment` / `view` |
| `password` | string | 否 | 密码房间使用 |
| `voiceEnabled` | boolean | 否 | 是否显示语音入口 |

响应:

```json
{
  "roomId": "demo1234",
  "name": "Demo room",
  "wsUrl": "ws://localhost:8088/ws/backend1/collab",
  "createdAt": 1780123000000,
  "accessMode": "link",
  "permissionMode": "edit",
  "passwordProtected": false,
  "voiceEnabled": false
}
```

### 2.7 历史恢复

```http
GET /api/rooms/{roomId}/history
GET /api/rooms/{roomId}/history?at={timestampMs}
```

响应:

```json
{
  "snapshot": {
    "snapshotId": "snap-1",
    "hlc": "",
    "createdAt": 1780123933635,
    "payload": "{\"shape-main\":{\"shapeType\":\"rect\",\"x\":{\"value\":1580,\"hlc\":\"1780123820639.1501.node-local\",\"writerId\":\"perf-user-a\"}}}"
  },
  "ops": [
    {
      "opId": "op-1",
      "userId": "user-a",
      "hlc": "1780123820639.1501.node-local",
      "createdAt": 1780123820640,
      "payload": "{\"opId\":\"op-1\",\"opType\":\"update\",\"shapeId\":\"shape-main\",\"shapeType\":\"rect\",\"attrs\":{\"x\":100}}"
    }
  ]
}
```

说明:

- 后端会查询 `createdAt <= at` 的最新 snapshot。
- `ops` 只返回 snapshot 之后到 `at` 之间的增量操作。
- snapshot payload 是 versioned snapshot, 每个属性可包含:
  - `value`
  - `hlc`
  - `writerId`

## 3. WebSocket 协议

### 3.1 连接地址

常用地址:

```text
ws://localhost:8088/ws/backend1/collab
ws://localhost:8088/ws/backend2/collab
```

兼容地址:

```text
ws://localhost:8088/ws/collab
```

客户端应优先使用 REST 房间查询返回的 `wsUrl`。

### 3.2 客户端到服务端

#### join

```json
{
  "type": "join",
  "msgId": "msg-1",
  "roomId": "demo1234",
  "userId": "user-a",
  "displayName": "Alice",
  "color": "#2563eb"
}
```

说明:

- WebSocket 建立后必须先发送 `join`。
- 服务端会把 session 加入房间, 并返回 `joined`。

#### cursor

```json
{
  "type": "cursor",
  "msgId": "msg-2",
  "roomId": "demo1234",
  "userId": "user-a",
  "x": 120.5,
  "y": 240.5
}
```

说明:

- 光标坐标使用画布世界坐标。
- 前端当前约 50ms 节流。
- 服务端广播给同房间其他客户端。

#### op

```json
{
  "type": "op",
  "msgId": "msg-3",
  "roomId": "demo1234",
  "userId": "user-a",
  "hlc": "1780123820639.1.user-a",
  "op": {
    "opId": "op-1",
    "opType": "create",
    "shapeId": "shape-1",
    "shapeType": "rect",
    "attrs": {
      "x": 100,
      "y": 120,
      "w": 140,
      "h": 90,
      "fill": "#93c5fd"
    }
  }
}
```

`opType`:

- `create`
- `update`
- `delete`

服务端处理顺序:

1. 合并 HLC。
2. 写入 MySQL 操作日志。
3. 写入成功后应用本节点内存副本。
4. 给发送者返回 `op-ack`。
5. 广播 `op` 给同房间其他客户端。
6. 通过 Redis Pub/Sub 分片频道广播给其他后端节点。

#### shape-preview

```json
{
  "type": "shape-preview",
  "msgId": "msg-4",
  "roomId": "demo1234",
  "userId": "user-a",
  "op": {
    "opId": "preview-1",
    "opType": "update",
    "shapeId": "shape-1",
    "shapeType": "pen",
    "attrs": {
      "points": [0, 0, 10, 10],
      "stroke": "#111827",
      "strokeWidth": 3
    }
  }
}
```

说明:

- 用于画笔绘制和拖拽预览。
- 当前仍走普通跨节点广播。
- 后续可降级为可丢弃低优先级消息。

### 3.3 服务端到客户端

#### joined

```json
{
  "type": "joined",
  "roomId": "demo1234",
  "you": {
    "userId": "user-a",
    "displayName": "Alice",
    "color": "#2563eb"
  },
  "peers": []
}
```

#### peer-joined

```json
{
  "type": "peer-joined",
  "userId": "user-b",
  "displayName": "Bob",
  "color": "#16a34a"
}
```

#### peer-left

```json
{
  "type": "peer-left",
  "userId": "user-b"
}
```

#### cursor

```json
{
  "type": "cursor",
  "userId": "user-b",
  "displayName": "Bob",
  "color": "#16a34a",
  "x": 120.5,
  "y": 240.5
}
```

#### op-ack

```json
{
  "type": "op-ack",
  "opId": "op-1",
  "hlc": "1780123820639.1501.node-local"
}
```

说明:

- 只发送给原始发送者。
- 表示该 op 已经持久化并应用。
- 前端收到后删除 pending outbox 中对应 op。
- 前端同时用服务端返回的 HLC 校准本地 CRDT 版本。

#### op

```json
{
  "type": "op",
  "userId": "user-a",
  "hlc": "1780123820639.1501.node-local",
  "op": {
    "opId": "op-1",
    "opType": "update",
    "shapeId": "shape-1",
    "shapeType": "rect",
    "attrs": {
      "x": 140,
      "y": 160
    }
  }
}
```

说明:

- 发送者不会收到自己的 `op` 广播, 发送者依赖 `op-ack`。
- 其他客户端收到后应用 CRDT 合并。

#### shape-preview

```json
{
  "type": "shape-preview",
  "userId": "user-a",
  "op": {
    "opId": "preview-1",
    "opType": "update",
    "shapeId": "shape-1",
    "shapeType": "pen",
    "attrs": {
      "points": [0, 0, 10, 10]
    }
  }
}
```

#### error

```json
{
  "type": "error",
  "code": "not_joined",
  "message": "Send join before op"
}
```

当前错误码:

| code | 说明 |
|---|---|
| `not_joined` | join 前发送 cursor/op/shape-preview |
| `unsupported_type` | 未支持的消息类型 |
| `op_persist_failed` | op 未能写入 MySQL, 不会广播 |

## 4. Redis Key 与频道

节点注册:

```text
cocanvas:nodes              Set, 保存 nodeId
cocanvas:nodes:{nodeId}     String, JSON NodeInfo, TTL 15 秒
```

Pub/Sub:

```text
cocanvas:room-events:{0..63}
```

分片规则:

- 对 `roomId` 做 SHA-256。
- 取前 4 字节计算 hash。
- `floorMod(hash, 64)` 得到频道编号。

## 5. MySQL 表

当前主要表:

- `rooms`
- `operation_logs`
- `snapshots`

`operation_logs`:

- `opId` 为主键。
- `roomId, createdAt` 有索引。
- `roomId, hlc` 有索引。

`snapshots`:

- `snapshotId` 为主键。
- `roomId, createdAt` 有索引。
