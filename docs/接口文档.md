# 接口文档

> 更新日期: 2026-06-02
> 说明: 本文档描述当前代码实际实现的 REST 和 WebSocket 协议。旧 `05_接口文档.md` 保留为历史设计材料, 当前接口以本文档为准。

## 1. 通用约定

基础地址:

```text
HTTP:      http://localhost:8088/api
WebSocket: ws://localhost:8088/ws/backend1/collab
WebSocket: ws://localhost:8088/ws/backend2/collab
```

开发兼容路径:

- 多数 REST 接口同时兼容去掉 `/api` 前缀的路径。
- 通用 WebSocket 路径 `/ws/collab` 仍保留。

数据格式:

- JSON。
- UTF-8。
- 字段使用 camelCase。
- 时间为毫秒时间戳。
- HLC 格式为 `physicalMs.logicalCounter.nodeId`, 示例: `1780123820639.1501.node-local`。

### 2.2 创建房间

```http
POST /api/rooms
Content-Type: application/json
```

可选请求头: `Authorization: Bearer <authToken>`。携带账号 token 创建房间时, 创建者会自动成为该房间 `owner`。

请求:

```json
{
  "roomId": "demo1234",
  "name": "Demo room",
  "accessMode": "link",
  "permissionMode": "edit",
  "password": "",
  "voiceEnabled": false
}
```

字段:

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `roomId` | string | 否 | 不传则后端生成短 ID |
| `name` | string | 否 | 房间名称 |
| `accessMode` | string | 否 | `link` / `password` / `private` |
| `permissionMode` | string | 否 | `edit` / `comment` / `view` |
| `password` | string | 否 | 密码房间使用 |
| `voiceEnabled` | boolean | 否 | 是否显示语音入口 |

响应:

```json
{
  "roomId": "demo1234",
  "name": "Demo room",
  "wsUrl": "ws://localhost:8088/ws/backend1/collab",
  "createdAt": 1780123000000,
  "accessMode": "link",
  "permissionMode": "edit",
  "passwordProtected": false,
  "voiceEnabled": false,
  "joinToken": "eyJ...",
  "memberRole": "owner"
}
```

### 2.7 历史恢复

```http
GET /api/rooms/{roomId}/history
GET /api/rooms/{roomId}/history?at={timestampMs}
```

响应:

```json
{
  "snapshot": {
    "snapshotId": "snap-1",
    "hlc": "",
    "createdAt": 1780123933635,
    "payload": "{\"shape-main\":{\"shapeType\":\"rect\",\"x\":{\"value\":1580,\"hlc\":\"1780123820639.1501.node-local\",\"writerId\":\"perf-user-a\"}}}"
  },
  "ops": [
    {
      "opId": "op-1",
      "userId": "user-a",
      "hlc": "1780123820639.1501.node-local",
      "createdAt": 1780123820640,
      "payload": "{\"opId\":\"op-1\",\"opType\":\"update\",\"shapeId\":\"shape-main\",\"shapeType\":\"rect\",\"attrs\":{\"x\":100}}"
    }
  ]
}
```

说明:

- 后端会查询 `createdAt <= at` 的最新 snapshot。
- `ops` 只返回 snapshot 之后到 `at` 之间的增量操作。
- snapshot payload 是 versioned snapshot, 每个属性可包含:
  - `value`
  - `hlc`
  - `writerId`

## 3. WebSocket 协议

### 3.1 连接地址

常用地址:

```text
ws://localhost:8088/ws/backend1/collab
ws://localhost:8088/ws/backend2/collab
```

兼容地址:

```text
ws://localhost:8088/ws/collab
```

客户端应优先使用 REST 房间查询返回的 `wsUrl`。

### 3.2 客户端到服务端

#### join

```json
{
  "type": "join",
  "msgId": "msg-1",
  "roomId": "demo1234",
  "userId": "user-a",
  "displayName": "Alice",
  "color": "#2563eb",
  "joinToken": "eyJ..."
}
```

说明:

- WebSocket 建立后必须先发送 `join`。
- 服务端会把 session 加入房间, 并返回 `joined`。
- `joinToken` 由创建房间或查询房间接口返回。
- token 默认 5 分钟有效。
- token 无效或过期时, 服务端返回 `invalid_join_token` 并关闭连接。

#### cursor

```json
{
  "type": "cursor",
  "msgId": "msg-2",
  "roomId": "demo1234",
  "userId": "user-a",
  "x": 120.5,
  "y": 240.5
}
```

说明:

- 光标坐标使用画布世界坐标。
- 前端当前约 50ms 节流。
- 服务端广播给同房间其他客户端。

#### op

```json
{
  "type": "op",
  "msgId": "msg-3",
  "roomId": "demo1234",
  "userId": "user-a",
  "hlc": "1780123820639.1.user-a",
  "op": {
    "opId": "op-1",
    "opType": "create",
    "shapeId": "shape-1",
    "shapeType": "rect",
    "attrs": {
      "x": 100,
      "y": 120,
      "w": 140,
      "h": 90,
      "fill": "#93c5fd"
    }
  }
}
```

`opType`:

- `create`
- `update`
- `delete`

服务端处理顺序:

1. 合并 HLC。
2. 应用本节点内存副本。
3. 给发送者返回 `op-ack`。
4. 广播 `op` 给同房间其他客户端。
5. 通过 Redis Pub/Sub 分片频道广播给其他后端节点。
6. 异步写入 MySQL 操作日志。

说明:

- 当前 `op-ack` 表示 op 已进入实时内存副本并完成 ack 关键路径。
- MySQL 持久化为批量写后置, 不阻塞 ack; 非优雅退出时尚未 flush 的 op 依赖周期 snapshot 兜底。

## 4. Redis Key 与频道

节点注册:

```text
cocanvas:nodes              Set, 保存 nodeId
cocanvas:nodes:{nodeId}     String, JSON NodeInfo, TTL 15 秒
```

Pub/Sub:

```text
cocanvas:room-events:{0..63}
cocanvas:room-transient-events:{0..63}
```

分片规则:

- 对 `roomId` 做 SHA-256。
- 取前 4 字节计算 hash。
- `floorMod(hash, 64)` 得到频道编号。
- op / peer joined / peer left / room-phase / room-phases / room-members 走 `room-events`。
- cursor / shape-preview / room-chat / room-emoji 走 `room-transient-events`。

## 5. MySQL 表

当前主要表:

- `users`
- `rooms`
- `room_members`
- `operation_logs`
- `snapshots`

`users`:

- `userId` 为主键。
- `username` 唯一。
- `passwordHash` / `passwordSalt` 保存 PBKDF2 账号密码凭据。

`room_members`:

- `(roomId, userId)` 唯一。
- `role` 为 `owner` / `edit` / `comment` / `view`。
- `invitedByUserId` 记录邀请或管理者。
- `updatedAt` 记录最后角色变化时间。

`operation_logs`:

- `opId` 为主键。
- `roomId, createdAt` 有索引。
- `roomId, hlc` 有索引。

`snapshots`:

- `snapshotId` 为主键。
- `roomId, createdAt` 有索引。
