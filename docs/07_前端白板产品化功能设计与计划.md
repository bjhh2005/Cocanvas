# 前端白板产品化功能设计与实现计划

## 1. 背景

当前 Cocanvas 已经具备多人实时协作白板的基础链路:

- 创建/加入房间
- WebSocket 多人同步
- 远端光标
- 矩形、圆形、文本的创建、拖动、删除
- HLC/CRDT 合并
- Redis 多节点广播
- MySQL 操作日志与历史恢复
- 一致性哈希路由与重连恢复

但从产品体验看,它仍然更像"协作同步技术 Demo",还不是一个真正可用的在线协作白板工具。下一阶段目标是把前端升级成面向真实使用场景的白板工作台。

本设计参考 Miro 等在线协作白板工具的公开功能形态,但不做完整复刻。Cocanvas 的重点仍然是课程项目中的协作、分布式同步和可演示性。

参考资料:

- Miro Sticky Notes: https://help.miro.com/hc/en-us/articles/360017572054-Sticky-Note
- Miro Essential Tools: https://help.miro.com/hc/en-us/sections/26264590241682-Essential-Tools
- Miro Mapping & Diagramming: https://help.miro.com/hc/en-us/articles/4403634496402-Miro-for-Mapping-Diagramming
- Miro Web SDK Board Items: https://developers.miro.com/docs/boards
- Miro Connector: https://developers.miro.com/docs/connector_connector-1

## 2. 产品目标

### 2.1 用户感知目标

用户进入房间后,第一眼应该感到这是一个完整的在线白板工作台,而不是一个简单画布 Demo。

核心体验:

- 可以用左侧工具栏快速创建便签、文本、形状、连接线和画笔。
- 可以拖动画布、缩放画布,在更大的空间里组织内容。
- 可以选中对象后快速修改颜色、字体、边框、层级。
- 可以多人同时编辑,并看到彼此光标与操作结果。
- 断线期间的操作可以先保留,重连后自动同步。
- 可以用历史能力恢复当前画布状态。

### 2.2 技术目标

- 继续复用现有 WebSocket、CRDT、HistoryService、RoomReplicaService。
- 不推翻现有 `shapeStore`,而是逐步演进为通用 `boardItemStore`。
- 前端功能优先产品化,后端协议按需扩展。
- 每一阶段都保持可运行、可演示。

## 3. 当前问题

### 3.1 功能太少

当前只有:

- Rect
- Circle
- Text
- Delete

缺少真实白板高频工具:

- 便签
- 连接线/箭头
- 画笔/自由绘制
- 多选
- 框选
- 复制/粘贴
- 对齐/层级
- 缩放/平移
- 样式编辑
- 评论
- 框架/分区
- 模板

### 3.2 画布空间有限

当前画布是一个固定尺寸区域,更像"画板组件",不是无限白板。

需要加入:

- pan
- zoom
- 坐标变换
- 背景网格
- 视口状态
- zoom 控件
- fit to content

### 3.3 工具交互不完整

当前对象只能创建、拖动、删除。

需要补充:

- 单选/多选
- 选择框
- 拖拽创建对象
- 选中态控制点
- 右键/浮动上下文工具条
- 快捷键
- inline text editing

### 3.4 数据模型过窄

当前 `ShapeOperation` 面向 shape:

```ts
{
  opType,
  shapeId,
  shapeType,
  attrs
}
```

后续应该抽象为 board item:

```ts
{
  opType,
  itemId,
  itemType,
  attrs
}
```

兼容方式:

- 短期保留 `ShapeOperation` 命名。
- 中期新增 `BoardOperation`。
- 后端可以先兼容两种字段,再逐步迁移。

## 5. 核心功能设计

## 5.1 无限画布与导航

### 功能

- 鼠标滚轮缩放。
- Space/Hand 工具拖动画布。
- 右下角显示当前 zoom 百分比。
- 提供 `+`、`-`、`Fit` 按钮。
- 画布背景用轻量网格。

### 状态

```ts
type ViewportState = {
  scale: number;
  offsetX: number;
  offsetY: number;
  tool: ToolMode;
};
```

### 实现建议

- Konva `Stage` 使用 `scaleX/scaleY` 和 `x/y` 实现视口变换。
- 鼠标坐标统一通过 `screenToCanvas()` 转换。
- 远端光标继续用屏幕坐标显示,或升级为 canvas 坐标后再按当前视口渲染。

### 验收标准

- 可以缩放画布。
- 可以拖动画布。
- 创建对象时位置落在当前可见画布中心。
- 拖动对象在缩放状态下坐标正确。

## 5.2 工具模式

### 工具列表

```ts
type ToolMode =
  | 'select'
  | 'hand'
  | 'sticky'
  | 'text'
  | 'shape'
  | 'connector'
  | 'pen'
  | 'comment'
  | 'frame';
```

### 行为

- Select: 选择、移动、框选、多选。
- Hand: 拖动画布。
- Sticky: 点击画布创建便签。
- Text: 点击画布创建文本。
- Shape: 展开形状菜单后创建图形。
- Connector: 从对象锚点拖出连接线。
- Pen: 自由绘制路径。
- Comment: 点击对象或画布添加评论标记。
- Frame: 拖拽创建一个框架区域。

### 验收标准

- 左侧工具栏可切换工具。
- 当前工具有明确选中态。
- Esc 返回 Select。
- V 切换 Select。
- H 切换 Hand。
- N 创建 Sticky。
- T 创建 Text。

## 5.3 便签 Sticky Note

Miro 中便签是白板高频对象,支持颜色、文字、尺寸和快速创建。Cocanvas 应优先实现。

### 数据结构

```ts
type StickyAttrs = {
  x: number;
  y: number;
  w: number;
  h: number;
  text: string;
  fill: string;
  textColor: string;
  fontSize: number;
  authorId?: string;
  authorName?: string;
};
```

### 功能

- 点击画布创建便签。
- 双击便签进入文本编辑。
- 支持颜色预设。
- 支持拖动。
- 支持 Delete/Backspace 删除。
- 支持复制。

### 颜色预设

先使用 8 个可读性较好的颜色:

- Yellow `#ffd966`
- Blue `#9fc5e8`
- Green `#b6d7a8`
- Pink `#ead1dc`
- Orange `#f9cb9c`
- Purple `#d9d2e9`
- Mint `#b7e1cd`
- White `#ffffff`

### 验收标准

- 多人创建/编辑/移动便签可同步。
- 断线创建便签会进入 pending 队列,重连后同步。
- 历史恢复能还原便签。

## 5.4 文本工具

### 功能

- 点击画布创建文本。
- 双击编辑。
- 支持字体大小。
- 支持加粗/斜体。
- 支持文本颜色。
- 支持对齐方式。

### 实现建议

Konva 原生文本编辑较弱,可采用:

- 正常渲染用 Konva `Text`。
- 编辑时在画布上方定位一个 HTML `textarea`。
- 失焦或 Enter 提交为 update op。

### 验收标准

- 文本可编辑。
- 编辑结果多人同步。
- 编辑过程中本地体验流畅。

## 5.5 形状库

### 初始形状

- Rectangle
- Rounded rectangle
- Circle
- Diamond
- Triangle
- Arrow block

### 样式

- Fill
- Stroke
- Stroke width
- Text inside shape

### 验收标准

- Shape 菜单可以选择具体形状。
- 创建后可以编辑颜色和文字。
- 拖动、删除、恢复、pending 队列都可用。

## 5.6 连接线与箭头

Miro 的 diagramming 能力核心是对象之间的 connector。Cocanvas 可以先做轻量版本。

### 数据结构

```ts
type ConnectorAttrs = {
  fromItemId: string;
  toItemId: string;
  fromAnchor?: 'top' | 'right' | 'bottom' | 'left' | 'center';
  toAnchor?: 'top' | 'right' | 'bottom' | 'left' | 'center';
  points?: number[];
  stroke: string;
  strokeWidth: number;
  arrowEnd: boolean;
  label?: string;
};
```

### 阶段 1

- 选中对象后显示四个连接点。
- 从连接点拖到另一个对象创建直线箭头。
- 对象移动时连接线自动跟随。

### 阶段 2

- 支持折线。
- 支持 label。
- 支持连接到画布空白位置。

### 验收标准

- 两个对象可以用箭头连接。
- 移动任一对象,连接线端点跟随。
- 多人同步连接线。

## 5.7 画笔与自由绘制

### 数据结构

```ts
type PenPathAttrs = {
  points: number[];
  stroke: string;
  strokeWidth: number;
};
```

### 功能

- Pen 模式按住鼠标绘制。
- 松手后生成一个 path item。
- 支持颜色和粗细。
- 支持删除。

### 性能策略

- 绘制过程中本地即时显示。
- 松手后一次性发送 create op。
- 不要每个 mousemove 都广播。

### 验收标准

- 可以绘制自由线条。
- 松手后其他客户端看到线条。
- 历史恢复能还原线条。

## 5.8 多选、框选与批量操作

### 功能

- Shift 点击多选。
- 空白处拖拽框选。
- 多选后一起拖动。
- 批量删除。
- 批量修改颜色。

### 实现建议

- 前端维护 `selectedIds: string[]`。
- 多选移动时对每个 item 发 update op。
- 大量移动时可节流。

### 验收标准

- 可以一次选中多个对象。
- 可以批量移动和删除。
- 远端最终状态一致。

### 单选工具

- Font size
- Duplicate

- Send backward

### 多选工具

- Align left

- Align right

- Delete

### 验收标准

- 选中对象后无需去左侧工具栏即可修改常用属性。

## 5.11 复制、粘贴、撤销、重做

### 复制粘贴

- Ctrl/Cmd + C 复制选中对象。
- Ctrl/Cmd + V 粘贴到视口中心或鼠标位置。
- 新对象生成新 ID。

### 撤销重做

短期只做本地撤销:

- 记录本用户发出的 op。
- Ctrl/Cmd + Z 发出反向 op。
- Ctrl/Cmd + Shift + Z 重做。

中期再做多人协作下的语义撤销。

### 验收标准

- 复制粘贴可用。
- 单用户撤销自己的上一步操作。

## 5.12 评论

```ts

  x: number;
  y: number;
  text: string;
  authorId: string;
  authorName: string;
  resolved: boolean;
};
```

### 验收标准

- 可以创建评论。
- 评论多人同步。
- 评论可以标记 resolved。

### 功能

- 拖拽创建 frame。
- frame 显示标题。

### 验收标准

## 6. 数据模型演进

## 6.1 短期兼容模型

继续使用 `ShapeOperation`,但扩大 `shapeType` 范围。

```ts
type ShapeType =
  | 'rect'
  | 'circle'
  | 'text'
  | 'sticky'
  | 'diamond'
  | 'triangle'
  | 'connector'
  | 'pen'
  | 'comment'
  | 'frame';
```

优点:

- 后端改动小。
- 可以复用现有 CRDT 和历史链路。

缺点:

- 命名不够准确。

## 6.2 中期通用模型

迁移为:

```ts
type BoardItemType =
  | 'shape'
  | 'sticky'
  | 'text'
  | 'connector'
  | 'path'
  | 'comment'
  | 'frame';

type BoardOperation = {
  opId: string;
  opType: 'create' | 'update' | 'delete';
  itemId: string;
  itemType: BoardItemType;
  attrs?: Record<string, unknown>;
  hlc?: string;
  writerId?: string;
};
```

迁移策略:

1. 前端先增加新 item type,仍走旧字段。
2. 后端 DTO 保持兼容。
3. 协议文档新增 BoardOperation。
4. 最后重命名 store 和 DTO。

## 8. 协作与 CRDT 策略

继续沿用属性级 LWW:

- 每个 item 的每个属性独立保存 `value + hlc + writerId`。
- delete 使用 tombstone。
- pending op 重放使用稳定 `opId`。

新增注意点:

- 文本编辑不要每个字符都发 op,可在输入期间本地编辑,失焦或短 debounce 后发送。
- Pen 绘制不要每个点都发 op,松手后一次 create。
- 多选移动会产生多个 update,需要节流或批量 op。
- Connector 端点跟随可以前端渲染时计算,不必每次对象移动都改 connector attrs。

## 11. 关键验收脚本

### 单人操作

1. 创建房间。
2. 缩放画布。
3. 平移画布。
4. 创建 sticky。
5. 编辑 sticky 文本。
6. 修改 sticky 颜色。
7. 创建文本。
8. 创建形状。

### 多人协作

1. 两个窗口进入同一房间。
2. A 创建 sticky,B 看到。
3. B 编辑 sticky,A 看到。
4. A 移动 sticky,B 看到。
5. B 修改颜色,A 看到。

### 断线恢复

1. A 进入房间。
2. A 断网。
3. A 创建 sticky、编辑文本、移动对象。
4. A 恢复网络。
5. A 显示 flushed pending ops。
6. B 进入同一房间后看到 A 断线期间的操作。

### 历史恢复

1. 创建多个对象。
2. 等待/触发快照。
3. 刷新页面。
4. 重连后自动恢复当前画布。

## 12. 风险

- Konva 内联文本编辑需要 HTML overlay,坐标变换必须准确。
- pan/zoom 会影响所有鼠标坐标,需要统一坐标工具函数。
- 多选移动可能产生大量 op,需要节流或批量协议。
- Connector 自动跟随对象移动要避免频繁写 attrs。
- 当前 store 名称仍是 shapeStore,短期可接受,中期应重命名。
