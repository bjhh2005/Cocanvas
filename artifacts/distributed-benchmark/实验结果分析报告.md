# 分布式基准测试结果分析报告

## 实验环境

本次结果仅对应当前这台本地实验机器和当前 Docker 部署环境，离开这个环境后，数值不应被当作统一上限。

### 宿主机信息

- 主机名：`LAPTOP-O2NF5ALD`
- 操作系统：`Windows`，内核版本 `10.0.26200`
- 架构：`x64`
- CPU：`11th Gen Intel(R) Core(TM) i5-1155G7 @ 2.50GHz`
- 逻辑核心数：`8`
- 物理内存：`7.79 GiB`
- 采集时空闲内存：`0.78 GiB`
- Node.js：`v22.14.0`

### 服务部署形态

- 网关：`nginx:alpine`
- 前端：Vite 开发服务，经 Nginx 暴露
- 后端节点：`backend1`、`backend2`
- 数据库：`mysql:8`
- 缓存 / 广播：`redis:7`
- 对外统一入口：`http://localhost:8088`
- WebSocket 路径：
  - `/ws/backend1/collab`
  - `/ws/backend2/collab`

### 环境说明

- 本轮测试在本地 Docker Desktop 环境中完成。
- 测试时系统可用内存较低，因此结果会受到宿主机资源压力影响。
- 该结果用于说明“这台机器上的本地可承载范围”，不代表跨机器、跨操作系统、跨资源配额都能复现同样数值。

## 本轮保留结果

本目录中保留的三份 `highest-stable.*` 文件对应的是当前探到的“最高稳定通过”配置：

- `routeRequests = 180`
- `sequentialOps = 220`
- `burstPeers = 14`
- `burstMessages = 120`
- `historyRequests = 14`

对应总工作量：

- 路由读取：`180` 次
- 可靠写入：`221` 次 websocket op
- 瞬时消息：`14 x 120 = 1680` 条
- 历史查询：`14` 次 `/history` + `14` 次 `/history/anchors`

## 核心结果

### 1. 路由与缓存

- 路由读取平均延迟：`5.32 ms`
- 路由读取 `p95`：`9.73 ms`
- 缓存命中率：`96.28%`
- `cacheDelta`：`96` 次请求里只有 `1` 次 miss

说明：
本地重复读取同一房间时，缓存命中已经比较高，HTTP 查询阶段不是当前主要瓶颈。

### 2. 多节点可靠写入

- 首次写入 ack：`1.83 ms`
- 首次跨节点 fanout：`3.41 ms`
- 可靠写入吞吐：`305.42 ops/s`
- ack `p95`：`7.83 ms`
- 最终跨节点消息送达：`通过`

说明：
在当前负载下，两节点之间的可靠写入和广播传播仍然稳定，没有出现明显排队或重传失控。

### 3. 瞬时消息压力

- 总 transient 消息数：`1680`
- burst 吞吐：`13094.96 msg/s`
- 队列峰值：`33`
- `transientDrops = 0`
- `overloadDisconnects = 0`

说明：
cursor / emoji 这类瞬时消息在该负载下还没有触发系统的保护阈值，说明消息发送队列仍可承受这一级别 burst。

### 4. 历史查询与最终一致性

- `history` 平均延迟：`29.78 ms`
- `history p95`：`43.50 ms`
- `anchors` 平均延迟：`14.30 ms`
- `history` 持久化操作数：`221`
- `history` 包含最终操作：`通过`
- `history` 包含最终文本：`通过`
- 重复读取结果稳定：`通过`

说明：
当前这组负载下，写入后的历史结果仍能在可接受时间内稳定读取出来，而且最终状态与最后一次写入匹配。

## 上限判断

本轮继续往上探时，下一档配置为：

- `routeRequests = 180`
- `sequentialOps = 240`
- `burstPeers = 15`
- `burstMessages = 120`
- `historyRequests = 14`

这一档在 `history` 轮询阶段失败，错误表现为：

- `history poll timed out before reaching 241 ops`

这说明当前环境下的边界不再只是 WebSocket 建连数，而是“高并发写入之后，历史链路在限定时间内完成落地和可见”的能力也开始成为限制项。

因此，本次实验将上一档稳定通过配置视为当前机器上的可复现实验上限。
