# 运行脚本说明 (`run.sh`)

## 1. 简介
`run.sh` 是 Cocanvas 后端环境的便捷管理脚本。它基于 Docker Compose，用于快速构建和启停 Spring Boot 后端服务及其依赖中间件（Redis 7 和 MySQL 8）。有了这个脚本，开发者无需在本地安装 Java 或 Gradle 即可完成常规的后端的运行和测试。

## 2. 常用命令

| 命令 | 说明 |
| --- | --- |
| `./run.sh up` | 构建并以后台模式启动所有服务（Spring Boot、Redis、MySQL）。 |
| `./run.sh app` | 仅构建并启动 Spring Boot（会自动关联启动依赖的服务）。 |
| `./run.sh deps` | 仅启动依赖环境（Redis 和 MySQL），不启动后端应用程序。 |
| `./run.sh test` | 在 Docker 容器内部运行后端的测试用例。 |
| `./run.sh down` | 停止并移除容器，**但是会保留 MySQL 数据卷**。 |
| `./run.sh clean` | 停止容器，并**彻底删除 MySQL 数据卷**（重置数据库使用）。 |
| `./run.sh logs` | 跟踪所有容器的日志输出。 |
| `./run.sh env` | 打印当前后端解析到的所有系统环境变量。 |

---

## 3. 环境变量参数说明（重点）

脚本支持通过环境变量来定制容器行为。**推荐方式**：基于 `.env.example` 复制一份 `.env` 文件到当前目录进行个人的参数配置。一旦由于端口占用导致启动失败，调整以下参数即可：

### 3.1 端口映射配置
为了防止和本地已有的服务冲突，所有暴露给宿主机（Host）系统的端口都可以自定义：
- `BACKEND_HOST_PORT`：后端的服务端口，默认为 `8080`。
- `REDIS_HOST_PORT`：Redis 的连接端口，默认为 `6379`。如果本地已安装 Redis 引发端口被占用报错，可修改（例如：改成 6380）。
- `MYSQL_HOST_PORT`：MySQL 数据库端口，默认为 `3307`。由于常见的 3306 极易冲突，所以脚本刻意改为了 3307。

### 3.2 Gradle 构建代理配置
当通过 Docker 容器内构建 Spring Boot 项目下载 Gradle 或相关依赖较慢时，脚本提供了代理加速注入：
- `GRADLE_PROXY_HOST`：Docker 构建时的 HTTP 代理地址，默认值为 `127.0.0.1`。
- `GRADLE_PROXY_PORT`：代理的端口号，默认值为 `26797`（如果你的本地代理端口是其他值，如 `7890`，请在此修改）。
> *注：如果你处于不需要网络代理的开发环境，可以通过执行 `GRADLE_PROXY_HOST= ./run.sh app` 来轻松清空并禁用代理配置。*

### 3.3 数据库初始化与项目隔离
- `MYSQL_ROOT_PASSWORD`：指定 MySQL 容器的 root 密码，默认为 `cocanvas123`。
- `MYSQL_DATABASE`：指定 MySQL 需要自动初始化的数据库名称，默认为 `cocanvas`。
- `COMPOSE_PROJECT_NAME`：Docker Compose 的项目前缀名称，默认为 `cocanvas`。如果需要启动多套隔离的环境，可以改变该值，此时生成的网络和数据卷也会自动隔离。
