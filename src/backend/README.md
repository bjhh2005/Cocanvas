# Cocanvas Backend

## Local configuration

Copy the example environment file before first run:

```bash
cp .env.example .env
```

Edit `.env` when a port conflicts on your machine.

Common values:

```bash
BACKEND_HOST_PORT=8080
REDIS_HOST_PORT=6379
MYSQL_HOST_PORT=3307
```

Container ports stay fixed. Only host ports are changed.
Container names are generated by Docker Compose from `COMPOSE_PROJECT_NAME`,
so changing the project name also isolates container, network, and volume names.

## Commands

```bash
./run.sh up       # build and start backend, Redis, and MySQL
./run.sh deps     # start only Redis and MySQL
./run.sh app      # build and start backend with dependencies
./run.sh ps       # show containers
./run.sh env      # show resolved environment values
./run.sh config   # show resolved Docker Compose config
./run.sh down     # stop and remove containers, keep MySQL data
./run.sh down-all # remove current and legacy backend compose containers
./run.sh clean    # remove containers and MySQL data volume
```
